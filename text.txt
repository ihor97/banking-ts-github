Мережа

OSI представлена для того щоб розмежувати зони відповідальності
-фізичний (це є кабель)
-канальниій рівень(все що стосується передачі сигналів(наприклад комутатор кабель який заєднує підїзд і компуктер))
макадрес-індивідуальний код по якому можна підключитися

-мережевий рівень -логічна адресація (марщутизатори) (використовує протоколи IP,IPSec)

-транспортний рівень - TCP UDP протоколи  Відповідає за забезпечення надійної доставки даних між вузлами. Розбиває дані на пакети, керує потоком даних та відновлює втрачені пакети.
TCP гарантує доставку пакетів 
UDP має вищу швидкість через те що там доставляються не всі пакети(наприклад просмотр відео)

-Сеансовий рівень- Відповідає за керування сеансами зв'язку між пристроями. Встановлює, управляє та завершує з'єднання між пристроями.

-Представлення - Відповідає за перетворення та кодування даних для забезпечення сумісності між різними системами. Включає стиснення, шифрування та форматування даних

Застосування (Application): Найвищий рівень, який відповідає за специфічні функції та послуги, доступні користувачам. Включає протоколи для різних додатків, таких як HTTP для веб-переглядачів або SMTP для електронної пошти.
взаємодія з кінцевим користувачем

Представники HTTP SSH DHCP(автоматично назначає айпішники в мережі)
